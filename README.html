Human review artifacts

Use HTML when the shape matters.

Markdown remains the working format for agent memory, specs, and durable notes. HTML is the review surface when a human needs layout, comparison, annotation, motion, or visual hierarchy instead of another wall of text.

Read the article Open DESIGN.md

Decision rule

Markdown for state. HTML for review.

Thariq Shihipar's HTML examples show agents producing planning boards, annotated PR reviews, design-system sheets, prototypes, diagrams, decks, explainers, reports, and custom editors as browser-openable HTML files.

Use Markdown when

  • The file is agent-readable memory, a spec, a task list, or a changelog.
  • The content should stay easy to diff, edit, and paste into prompts.
  • The output will be consumed by another agent or automation.

Use HTML when

  • The output is for a human reviewing complex information.
  • Spatial layout, color, severity, side-by-side comparison, or interaction helps.
  • The artifact should open directly in a browser without a dev server.

Token budget

Externalize decorative CSS.

The expensive part of attractive HTML is repeated styling. Future artifacts should link this shared stylesheet, then spend tokens on semantic content.

01 Link CSS

Use <link rel="stylesheet" href="./styles.css">.

02 No inline theme

Avoid large <style> blocks and repeated visual tokens.

03 Semantic shell

Use shallow sections, cards, grids, badges, diffs, and timelines.

04 Open locally

Keep artifacts static and browser-openable without a build step.

Project workflow

Fetch design once. Reuse the sheet.

This project uses the BMW M template from getdesign as the visual source, then converts the durable design choices into static CSS classes.
