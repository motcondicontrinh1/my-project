Implementation Plan · Revision Alpha

Rolling door, family-safe.

An ESP32 + 4-channel relay simulates the wall-panel buttons — Mở, Dừng, Đóng — so a phone or PWA can open and close the rolling door without ever touching the motor, capacitor, or 220V mains. The original RF remote and physical buttons keep working.

ESP32 4-ch relay HiveMQ Cloud React PWA Vercel

Safety principle

Never touch the motor side.

The ESP32 only closes a dry contact in parallel with the existing wall buttons. It is electrically isolated from the door motor, the capacitor, and the 220V mains.

Hard rule

Do not wire the ESP32 or relay directly to the motor, capacitor, mains 220V, or motor power lines. The relay must only bridge the existing wall-panel button terminals using COM + NO.

Why it stays safe

Relay contacts sit in parallel with the existing wall buttons. If the ESP32 is offline, unplugged, or rebooting, the wall panel and RF remote continue to work normally.

Firmware sets every relay pin to OFF (HIGH for low-trigger) immediately on boot, so no relay can be stuck ON after a power-loss event.

Wiring · GPIO mapping

One mapping, used everywhere.

Firmware, wiring diagrams, and documentation must all use this exact mapping. Inconsistent mapping is the most common source of installation mistakes.

Function ESP32 GPIO Relay input Relay contact Wall-panel button
Mở / LênGPIO16IN1Relay 1 COM + NOOpen / Up
DừngGPIO17IN2Relay 2 COM + NOStop
Đóng / XuốngGPIO18IN3Relay 3 COM + NOClose / Down
Spare—IN4Relay 4Leave empty

Relay power

ESP32 5V / VIN  → Relay VCC
ESP32 GND       → Relay GND

Low-trigger logic

GPIO HIGH → relay OFF
GPIO LOW  → relay ON

Firmware must drive all relay pins HIGH (OFF) immediately on boot.

MQTT topics

Private prefix, three topics.

Avoid generic public-looking topics. Use a unique prefix so opportunistic listeners on the broker cannot guess the device.

Command

home/hoa-door-7f3a/command

Allowed payloads:

OPEN
STOP
CLOSE

Device status

home/hoa-door-7f3a/device/status

Suggested payloads:

ONLINE
OFFLINE
MQTT_CONNECTED
MQTT_DISCONNECTED

Last command

home/hoa-door-7f3a/device/last-command

Suggested payloads:

OPEN_SENT
STOP_SENT
CLOSE_SENT

Until a physical door sensor is installed, the app must say “Lệnh gần nhất” / “Last command” — never “Cửa đang mở/đóng”. The system knows what it sent, not what the door actually did.

Implementation tasks

Twelve steps, in order.

Tasks 0 through 11. The real door is connected only after the ESP32 and relay behavior are already proven on the bench.

Task 0

Prepare tools and safety checklist

Gather ESP32, relay module, jumpers, USB charger, multimeter, terminal blocks, electrical box. Photograph the original wall-panel wiring before touching anything. Identify each button pair with continuity. Never assume wire color means function. Never connect to motor or 220V lines.

Task 1

ESP32 local web app

Arduino IDE + ESP32 board package. ESP32 hosts an Access Point at http://192.168.4.1 with three buttons (MỞ / LÊN, DỪNG, ĐÓNG / XUỐNG) calling routes /open, /stop, /close. Each route pulses one GPIO for 500 ms.

Test without relay: Serial Monitor logs the correct command for each button.

Task 2

Relay bench test

Wire ESP32 5V/GND to relay VCC/GND. GPIO16 → IN1, GPIO17 → IN2, GPIO18 → IN3. Press each web button and confirm exactly one relay clicks for ~500 ms. No two relays ON at the same time.

Task 3

Identify wall-panel button terminals

Power off the rolling-door system. Photograph existing wiring. Use multimeter continuity to find the two terminals that connect only when each button is pressed. Mark OPEN_A/OPEN_B, STOP_A/STOP_B, CLOSE_A/CLOSE_B. Do not remove existing wires unless absolutely necessary.

Task 4

Connect relay dry contacts to wall panel

Wire each relay's COM + NO in parallel with the matching button. Closing COM + NO is the same as a momentary press of the original button. Use COM + NO, never NC.

Task 5

Local full-system test

Confirm all relays OFF after boot. Verify physical wall buttons and RF remote still work. Then exercise OPEN, STOP, CLOSE from the local web page and confirm direction mapping. Pass condition: wall panel works, RF remote works, web buttons work, no wrong direction, no relay stuck ON.

Task 6

Set up HiveMQ Cloud broker

Create a free serverless cluster. Note broker host, port 8883 (ESP32 MQTTS), port 8884 (PWA WSS), username, password. Use a unique topic prefix such as home/hoa-door-7f3a — never door/command.

Task 7

ESP32 firmware — Wi-Fi + MQTT

WiFiClientSecure + PubSubClient on port 8883. Subscribe to the command topic, publish ONLINE on connect, add reconnect logic, keep the local web page as fallback. Test with MQTTX publishing OPEN / STOP / CLOSE.

Task 8

React PWA setup

Vite React project with one screen: Mở cửa (green/blue), Dừng (yellow/orange, large and central), Đóng cửa (red). Show device Online/Offline, MQTT Connected/Disconnected, last command. Confirm OPEN and CLOSE before sending. Add a UI PIN gate before showing controls.

Task 9

React PWA MQTT connection

Install mqtt. Connect WSS on port 8884. Publish to home/hoa-door-7f3a/command. Subscribe to status and last-command. Show connection state and last command in the UI.

Limit: direct PWA → MQTT exposes credentials in the bundle. Acceptable for early testing only.

Task 10

Deploy PWA to Vercel

Push to GitHub, connect to Vercel, configure env vars, deploy. Open the URL on family phones, Add to Home Screen. Test commands while on mobile data.

Task 11

Final hardware installation

Mount ESP32 + relay in a plastic electrical box. Use terminal blocks / Wago / domino — no loose Dupont wires for permanent joins. Power from a USB 5V 2A charger. Confirm reconnect after power loss and that all relays are OFF after reboot. Close the box only after full testing.

Future improvements

After the family is happy.

Out of scope for v1. Listed in rough priority order.

  • Magnetic reed switch on the door rail to detect real open/closed state.
  • Activity log of every command received and pulse fired.
  • Per-user authentication so each family member has their own credentials.
  • Backend API so MQTT credentials are no longer exposed in the frontend bundle.
  • Push notification when the door is left open beyond a timeout.
  • Auto-close timer with a safety confirmation step.
  • Physical emergency-stop input wired to the ESP32.
  • Watchdog timer and fail-safe reboot handling.
